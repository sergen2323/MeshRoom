# MeshRoom

Аналог Radmin VPN / Hamachi **без внешних серверов**: участники сами являются хостами.
Комнаты с UUID, приглашение по ссылке, чат, локальный профиль без регистрации и
WireGuard-туннель между машинами. Windows + macOS (+ Linux для разработки).

## Как это работает

- **Профиль** — локальный, без регистрации: ник, аватар, пароль (argon2id).
  Приватный ключ WireGuard шифруется паролем (AES-GCM) и хранится только на этом ПК.
- **Комната** — её создатель поднимает у себя control-сервис (TCP, шифрование
  ChaCha20-Poly1305 от секрета комнаты): реестр участников, выдача виртуальных IP
  из `100.64.0.0/10`, чат с историей, обмен WG-эндпоинтами.
- **Приглашение** — ссылка `meshroom://join?...` с UUID, секретом и адресами хоста.
- **Туннель** — wireguard-go (utun на macOS, Wintun на Windows). Создание адаптера
  требует прав администратора, поэтому туннелями управляет привилегированный
  помощник (`-helper`), а UI запрашивает права системным диалогом при включении.
- **Интерфейс** — локальный веб-UI (стекломорфизм, RU/EN), отдаётся самим бинарником
  на `127.0.0.1` и открывается в браузере.

Ограничение модели: комната доступна, только пока её хост в сети (как игровое лобби).

## Этап 1 — что готово

Профиль, комнаты, приглашения, чат, список участников с онлайн-статусом,
переключатель туннеля, UI.

## Этап 2 — что готово (интернет и надёжность)

- **STUN** (`internal/nat/stun.go`) — определение внешнего IP:port с WG-порта
  через публичные STUN-серверы.
- **Проброс порта** (`internal/nat`) — NAT-PMP (RFC 6886) и UPnP-IGD
  (SSDP + SOAP AddPortMapping), автопродление аренды.
- **Кандидаты endpoint** — LAN-адреса + внешний STUN-адрес, обмен через
  control-канал; WireGuard сам выбирает рабочий путь.
- **Relay-fallback** (`internal/relay`) — когда прямой путь между двумя
  участниками не установить, WG-трафик идёт через relay хоста комнаты
  (кастомный `conn.Bind` для wireguard-go). Хост заведомо достижим, а сам
  трафик остаётся зашифрованным Noise-протоколом WireGuard — relay видит
  только «конверты».
- **Диагностика в UI** — индикатор соединения с каждым участником
  (прямое / через ретранслятор / офлайн) и подсказка хосту, если порт
  недоступен снаружи.

## Этап 3 — установщики и CI

- **Windows** — `build/windows/meshroom.iss` (Inno Setup 6): ставит
  `meshroom.exe` + подписанный `wintun.dll` с wintun.net, ярлыки, деинсталлятор.
- **macOS** — `build/macos/make-app.sh`: универсальный (arm64+intel)
  `MeshRoom.app` с иконкой и ad-hoc подписью, упакованный в `.dmg`.
- **CI** — `.github/workflows/release.yml`: пуш тега `v*` собирает оба
  установщика и прикладывает их к GitHub-релизу автоматически.
- Иконки генерируются кодом: `build/gen-icon` (артворк PNG),
  `build/gen-ico` (Windows .ico), `build/gen-icns` (macOS .icns) — без
  внешних графических инструментов.

### Выпуск релиза

```sh
git tag v0.2.0 && git push origin v0.2.0
# через ~5 минут установщики появятся в GitHub → Releases
```

Локальная сборка macOS-версии: `build/macos/make-app.sh 0.2.0 universal`
(результат в `dist/`).

## Сборка и запуск

Требуется Go 1.23+.

```sh
# из каталога проекта
go build -o meshroom ./cmd/meshroom
./meshroom            # откроет UI в браузере на http://127.0.0.1:8790
```

Флаги: `-port N` (порт UI), `-no-browser` (не открывать браузер), `-helper`
(внутренний режим привилегированного помощника, вручную запускать не нужно).

## Проверка «две машины»

1. Запусти MeshRoom на ПК-A, создай профиль, создай комнату, скопируй ссылку.
2. На ПК-B запусти MeshRoom, создай свой профиль, «Войти по ссылке».
3. На обеих машинах включи туннель (тумблер) — появится системный запрос прав.
4. Пингуй виртуальный IP другой машины (`ping 100.x.y.z` из списка участников).
5. Чат работает независимо от туннеля.

В той же сети связь установится напрямую (индикатор «прямое соединение»).
Через интернет: если у хоста открыт порт (UPnP/NAT-PMP — плашка «порт открыт»),
участники подключатся напрямую; иначе трафик пойдёт через ретранслятор хоста
(индикатор «через ретранслятор») — медленнее, но работает без ручной настройки.

## Статус по коду

| Модуль | Назначение |
|---|---|
| `internal/identity` | профиль, пароль, ключи WG |
| `internal/proto` | сообщения control-канала, шифрование, IPAM, приглашения |
| `internal/room/host` | сервис хоста комнаты |
| `internal/room/client` | подключение участника, переподключение |
| `internal/tunnel` | wireguard-go, привилегированный помощник, per-OS настройка адаптера, relay-Bind |
| `internal/nat` | STUN, NAT-PMP, UPnP-IGD, автопроброс порта |
| `internal/relay` | UDP-ретранслятор хоста + relay-aware conn.Bind |
| `internal/app` | ядро: связывает всё + HTTP API и SSE для UI |
| `web/` | интерфейс (HTML/CSS/JS, стекломорфизм, i18n) |
